# 🚀 SCM Platform Deployment Guide

This guide describes how to deploy the SCM Platform online:
1. **Backend & Database** on **Render** (via Render Blueprints)
2. **Frontend** on **Vercel**

---

## 1. Deploy the Backend & Database to Render

The repository includes a `render.yaml` Blueprint file that automatically provisions a managed PostgreSQL database and builds the Spring Boot backend container.

### Deploy Steps:
1. Click the button below to start deployment on Render:
   
   [![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/jeeva1101/Vaizai-SCM)

2. **Sign in/Log in** to your Render account.
3. In the **Blueprint Configuration** page:
   - Enter a **Service Group Name** (e.g., `vaizai-scm`).
   - Leave all other default settings as they are.
4. Click **Apply**.
5. Render will provision the PostgreSQL database and compile the Spring Boot application using the root `Dockerfile`.
6. Once the deployment finishes, copy your **Web Service URL** (e.g., `https://vaizai-scm-backend.onrender.com`). You will need this for the frontend Vercel deployment.

---

## 2. Deploy the Frontend to Vercel

Vercel is used to host the Angular SPA.

### Deploy Steps:
1. Click the button below to start deployment on Vercel:

   [![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/jeeva1101/Vaizai-SCM)

2. **Select your Git provider** and import the `Vaizai-SCM` repository.
3. In the **Configure Project** step, make the following configuration changes:
   - **Framework Preset**: Select `Angular` (or it will auto-detect).
   - **Root Directory**: Click *Edit* and select the **`frontend`** directory (this is crucial since this is a monorepo).
4. Expand the **Environment Variables** section and add the following variable:
   - **Key**: `API_URL`
   - **Value**: Your Render Backend Web Service URL followed by `/api` (e.g., `https://vaizai-scm-backend.onrender.com/api`).
5. Click **Deploy**.
6. Vercel will run the custom build command (`node scripts/set-env.js && ng build`) to inject the live API URL and build the production bundle.
7. Once finished, open your new Vercel site URL to use the fully operational online application!

---

## 🛠️ How It Works Under The Hood

- **Dynamic Port Binding**: The backend's `application.yml` maps `server.port` dynamically to the `${PORT}` environment variable allocated by Render.
- **Database Connection**: `ScmApplication.java` automatically intercepts the `DATABASE_URL` PostgreSQL connection string generated by Render on startup, parses the connection parameters, and configures the Spring JPA dataSource dynamically.
- **Automated Seeding**: On the first start, `DatabaseInitializer.java` detects an empty database and runs the `data.sql` seeding script to create demo profiles (like Elizabeth Admin, John Procurement). Subsequent deployments skip seeding to preserve user modifications.
- **CORS Handling**: `CorsFilter.java` automatically whitelists and accepts CORS requests coming from any `.vercel.app` domain.
